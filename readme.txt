Las decisiones para el desarrollo de esta aplicación se basaron en la elección de
la API de Rick and Morty, debido a su amplia cantidad de datos y facilidad de implementación,
respaldada por una documentación clara con ejemplos para Flutter.

Se optó por una navegación y manejo de estados nativos para mantener la simplicidad.
Además, la arquitectura sigue el patrón MVC con componentes con sus respectivos servicios,
garantizando escalabilidad y reutilización del código.
